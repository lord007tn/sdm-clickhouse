# v0.1.4 Execution Plan

Date: 2026-02-26

## Phase 1: Updater hardening

Goals:

- Remove "no endpoints" native updater failures by setting a default endpoint.
- Keep GitHub fallback path for repos without updater manifest.
- Make update URL validation strict and safe.
- Make default updater repo configurable and documented.

Acceptance:

- `Check for updates` no longer fails with endpoint-missing error.
- Fallback update path still works when endpoint is unavailable.
- Download URL host validation rejects non-GitHub hosts.

## Phase 2: UX improvements

Goals:

- Reduce fragile prompt/confirm interactions for common file operations.
- Improve accessibility (aria labels, semantic buttons, label/input associations).
- Keep advanced connection options hidden unless toggled.

Acceptance:

- Export/import/backup/restore use native file dialogs instead of prompt paths.
- Icon-only actions have `aria-label`.
- Connection list is keyboard accessible.

## Phase 3: Developer experience

Goals:

- Add linting and formatting checks.
- Add at least one automated UI smoke test.
- Enforce checks in CI.

Acceptance:

- `pnpm lint` and `pnpm format:check` exist and pass.
- Playwright smoke test runs in CI.

## Phase 4: Validation and release readiness

Goals:

- Re-run type/build/rust/test/install checks.
- Document any residual limitations clearly.

Acceptance:

- Validation commands pass locally.
- Changelog/readme reflect behavior changes.
